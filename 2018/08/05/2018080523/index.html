记解码耳放一体机“SINC”的制作（长期更新中）

发表于 2018-08-05 更新于 2020-04-15 分类于 电路与系统

大学本科的时候就一直想要做一台自己的解码耳放一体机。自从做完邀请赛，便下了决心画板是不可能画板的，这辈子都不可能画板的。结果是管不住自己的手，没过多久就为同学毕设交差画了一块板，现在又禁不住阿纲的唆使，额:(

  上个月阿纲疯狂采购了3D打印机、焊台等一些工具，之后就问我要不要画板，想了想算了。前段日子老板叫我用visio给她画几张高频课用的电路图，还帮她做了一份教学用的幻灯片，后来听说她在全国授课竞赛上拿了奖，看样子作为好队友的我没有坑。经不起阿纲的唆使，我也没挑黄道吉日，瞎定了个方案。想起我做的第一个耳放“RX-1”也是唯一的一个耳放，它底噪清晰、高音啸叫、低频没有、中频扭曲、还带有不知道从哪里串进来的清脆打鼓声，白瞎了我两颗OPA627，毕业的时候直接踩碎扔垃圾桶，因此我拒绝再用RX作为前缀:(。近来实验室在圈子里越来越广为人知，我也蹭一蹭热度，就把这台机器拟叫做“SINC”，取sinc函数之意。

音频与控制方案

  先简单介绍一下音频与控制部分的方案：

功能 器件
IIS COMBO384+ISO7640
DAC PCM1792A
I/V OPA1612
Head Amp TPA6120A2
MCU STM32F103C8T6
OLED 0.96 OLED with SPI/I2C

  USB界面是采用的COMBO384官方方案，COMBO384也是一代神界面，被烧友们吹上了天，玩不懂XMOS又不想画CM6631的板子，最后就选了这个方案。DAC开始准备用WM8741，比较便宜，后来又想用CS4398，想了一下电压输出的DAC不知道怎么玩，最后选了电流输出的1792。都说1792是DAC之王，又说声音和DAC没啥关系，主要看电路，我想算了算了，直接抄官方电路省事，就直接拿6120的电路上了。又听说魅族用了OPA1612这枚运放，想必也是非常Hi-Fi，所以I/V变换就用它了。1792是个软控DAC，用起来比较麻烦，1794又没申请到，干脆就用32做软控了。耳放还需要音量调节，6120这里做了差分转单端，所以音量在它身上搞音量不合适，就拿32读旋转编码器调音量了，最后显示在OLED上。
